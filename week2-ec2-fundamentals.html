Understanding EC2

What is EC2?

Elastic Compute Cloud

EC2 gives you virtual servers (called instances) that you can start, stop, and scale on demand. Instead of buying physical hardware, you rent computing power by the second.

Key Concept: EC2 is IaaS (Infrastructure as a Service). You control the OS, the software, the networking. AWS handles the physical hardware underneath.

What You Choose

OS — Linux, Windows, macOS

Size — CPU, RAM, Storage

Network — Speed, public IP, placement

What AWS Handles

Hardware — servers, racks, cooling

Hypervisor — Nitro System

Physical Security — data centers

Naming Convention
# Example: m5.2xlarge

m    → Instance family (General Purpose)
5    → Generation (newer = better)
2xlarge → Size (CPU + RAM)

# Sizes (small → large):
nano → micro → small → medium
→ large → xlarge → 2xlarge → ...

# Common for dev:
t2.micro → Free Tier eligible
t3.small → Good for small apps
Exam Tip: t2.micro is Free Tier eligible (750 hours/month). Know the family letters and what they optimize for.
Network Security

Security Groups

What Are They?

Security Groups are virtual firewalls for your EC2 instances. They control what traffic can come in (inbound) and go out (outbound).

They only contain ALLOW rules. You cannot write a DENY rule in a Security Group.

Key Behaviors

Stateful — if traffic is allowed in, the response is automatically allowed out

Default — all inbound blocked, all outbound allowed

Multiple — one instance can have multiple SGs, one SG can be on multiple instances

Security Group Rules Example
# Inbound Rules (who can reach us)

Type        Port   Source
SSH         22     My IP only
HTTP        80     0.0.0.0/0 (everyone)
HTTPS       443    0.0.0.0/0 (everyone)
Custom      8080   sg-xxxxx (another SG)

# Outbound Rules (where we can go)

Type        Port   Destination
All traffic All    0.0.0.0/0 (default)
Exam Tip: If your app has a timeout error — it's a Security Group issue. If you get a "connection refused" — the app itself has a problem.
Advanced Feature

SG Referencing Other SGs

Instead of allowing specific IPs, you can allow another Security Group

This means: "any instance that has SG-B attached can talk to instances with SG-A". No need to manage IP addresses. Instances can change IPs and it still works.

EC2 Instance A
SG-Web (port 80)
←
allows SG-App
EC2 Instance B
SG-App (port 8080)
←
allows SG-DB
RDS Database
SG-DB (port 3306)
Real Example: Your web server (SG-Web) allows traffic from anyone on port 80. Your app server (SG-App) only allows traffic from SG-Web. Your database (SG-DB) only allows traffic from SG-App. This creates a secure chain — the internet can never reach your database directly.
Connecting

How to Access Your Instance

SSH (Linux/Mac)

The classic way. Use your terminal with a .pem key file.

$ ssh -i key.pem \
  ec2-user@public-ip
Make sure port 22 is open in your Security Group

EC2 Instance Connect

Browser-based SSH. No keys needed. Works directly from the AWS Console.

Still needs port 22 open, but AWS manages the temporary key for you

Session Manager (SSM)

No SSH, no port 22, no key files. Connects through AWS Systems Manager.

Most secure option. No inbound ports needed. Requires IAM Role with SSM permissions.
Important: Never use port 22 with source 0.0.0.0/0 in production. This opens SSH to the entire internet.
Pricing

EC2 Purchasing Options

On-Demand

Pay by the second. No commitment. Highest price but most flexible.

Use for: Short workloads, unpredictable traffic, testing

Reserved (1 or 3 years)

Up to 72% discount. You commit to a specific instance type in a specific region.

Use for: Steady-state apps, databases, predictable usage

Spot Instances

Up to 90% discount. But AWS can take it back with 2 min warning if they need the capacity.

Use for: Batch jobs, data analysis, workloads that can handle interruption

Dedicated Hosts

A physical server only for you. Most expensive. For compliance or licensing requirements.

Use for: Regulatory requirements, server-bound software licenses
Storage

EC2 Instance Storage

EBS (Elastic Block Store)

Network-attached storage. Like a USB drive you can plug into any instance.

Persists after instance stops/terminates (if configured)
Locked to one Availability Zone
Can be detached and attached to another instance

Instance Store

Physical disk attached to the server. Extremely fast but temporary.

Best I/O performance
Data LOST when instance stops or terminates
Good for caches, buffers, temp data

EBS Snapshots

Backup of your EBS volume at a point in time. Can copy across AZs and regions.

Create new volumes from snapshots
Move data between AZs using snapshots
Snapshot Archive: 75% cheaper, 24-72h to restore
EBS Deep Dive

EBS Volume Types

gp3 / gp2

General Purpose SSD. Balances price and performance.

gp3: 3,000 IOPS baseline, up to 16,000
gp2: IOPS scales with size (3 IOPS/GB)

io1 / io2

Provisioned IOPS SSD. Highest performance for critical workloads.

Up to 64,000 IOPS. Supports EBS Multi-Attach (multiple EC2s).

st1

Throughput Optimized HDD. Low cost for frequently accessed data.

Big data, data warehouses, log processing. Cannot be boot volume.

sc1

Cold HDD. Lowest cost. For infrequently accessed data.

Archive storage, lowest cost. Cannot be boot volume.
Exam Tip: Only gp2/gp3 and io1/io2 can be used as boot volumes. HDD types (st1, sc1) cannot boot an instance.
Machine Images

AMI — Amazon Machine Image

What is an AMI?

A template for your EC2 instance. It captures the OS, software, configuration — everything. Launch 100 identical instances from one AMI.

Three Sources

AWS Provided — Amazon Linux, Ubuntu, Windows
Marketplace — Third-party pre-configured AMIs
Custom — You build your own from an instance

Key Takeaways

Remember These for the
Exam

Security Groups = Stateful

Allow rules only. Return traffic is automatic. Timeout = SG issue, Connection refused = app issue.

EBS is AZ-Locked

To move an EBS volume, snapshot it first, then create a new volume in the target AZ.

Instance Store = Ephemeral

Highest performance but data is lost on stop/terminate. Good for caches only.

Only SSD Can Boot

gp2, gp3, io1, io2 can be boot volumes. st1 and sc1 (HDD) cannot.

Spot = Cheapest, Can Be Taken

Up to 90% off. 2-minute warning before termination. Only for fault-tolerant workloads.

AMI = Region-Specific

Must copy to use in another region. Custom AMIs speed up boot time significantly.
